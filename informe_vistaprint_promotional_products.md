## Blueprint para una implementación propia

- Diseñar una taxonomía estable: categoría → subcategoría → listado → producto.
- Dar a cada intención importante una URL indexable y enlazarla desde hubs relevantes.
- Usar una sola H1, secciones H2/H3 coherentes y breadcrumbs visibles con datos estructurados.
- Renderizar metadatos, contenido principal y enlaces críticos en HTML accesible para crawlers.
- Crear componentes propios para hero, category grid, product rail, use-case banner, trust block, FAQ y footer.
- Mantener canonicals autorreferentes, hreflang recíproco si hay países/idiomas y sitemaps segmentados.
- Optimizar imágenes propias (AVIF/WebP, srcset, dimensiones, lazy bajo el fold) y Core Web Vitals.
- Evitar copiar textos, imágenes, marcas, estilos exactos o código del sitio analizado.

## Limitaciones

- El análisis se basa en HTML público recibido por HTTP; contenido que sólo aparece tras interacción puede no figurar.
- Las tecnologías se infieren por firmas y pueden producir falsos positivos.
- Los tiempos registrados no sustituyen una medición Lighthouse/Core Web Vitals en navegador.
- El informe es una instantánea y puede cambiar por geolocalización, experimentos A/B o personalización.
